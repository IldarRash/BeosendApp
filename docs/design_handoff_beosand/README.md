# Handoff: BeoSand — применение дизайна к приложению

> Файл для локального Claude Code CLI. Описывает, **как перенести готовый визуальный язык**
> (две HTML-прототипа из дизайн-проекта) в реальный код **`apps/admin`** и **`apps/miniapp`**.
> Это инструкция по реализации, а не код для копипаста «как есть».

---

## 0. Что в этом бандле

| Файл | Что это | Как использовать |
|---|---|---|
| `tokens/admin-theme.css` | Полный набор дизайн-токенов админки (палитра, шрифты, геометрия, focus, тени, скроллбар) | **Скопировать содержимое `:root` в `apps/admin/src/ui/theme.css`** |
| `tokens/admin-components.css` | CSS всех компонентов админки со всеми состояниями | Разнести по `apps/admin/src/ui/*` (по компоненту на файл) |
| `tokens/miniapp-theme.css` | Токены и классы мини-аппа (Telegram-light + коралловый акцент) | Влить в `apps/miniapp/src/ui/theme.css` поверх `@telegram-apps/telegram-ui` |
| `Admin Console.html` | Эталон-прототип админки: 13 экранов + логин | Открыть в браузере — это источник истины по виду и поведению |
| `Mini App.html` | Эталон-прототип мини-аппа: онбординг + 8 экранов | Открыть в браузере (в device-рамке) |

**Фиделити: высокий (hi-fi).** Цвета, типографика, отступы и состояния финальные —
воспроизводить пиксель-в-пиксель средствами существующего стека (React + Vite,
дизайн-система в `src/ui/*`), а не верстать заново на глаз.

**Главное правило проекта сохраняется:** UI — только слой взаимодействия.
Никаких вычислений цен/доступности/вместимости на клиенте — дизайн **показывает готовые значения
сервера** и собирает действия. Все суммы — целые RSD, формат `1 500` (узкий неразрывный пробел-разряд).

## 1. Дизайн-токены админки

Вставить в `apps/admin/src/ui/theme.css` (полная версия — в `tokens/admin-theme.css`):

```css
:root {
  /* Песочные фоны */
  --sand-50:#FAF6EF; --sand-100:#F2EADD; --sand-200:#E7DBC7; --sand-300:#D8C9AF;
  /* Чернила (текст) */
  --ink-900:#211C15; --ink-700:#4B4339; --ink-500:#8A8073; --ink-400:#B0A695;
  /* Коралл (акцент) */
  --coral:#E15B43; --coral-ink:#B23E2B; --coral-soft:#FBE5DF;
  /* Бирюза (успех) */
  --teal:#1C9488; --teal-ink:#126B62; --teal-soft:#DBF0ED;
  /* Янтарь (warn/блокировки) */
  --amber:#C98A12; --amber-ink:#93630A; --amber-soft:#F7EBCF;
  /* Поверхности */
  --paper:#FFFFFF; --border:#E3D9C8; --border-strong:#CFC1A8;
  /* Фокус (доступность) */
  --focus:#2C6FB0; --focus-ring:0 0 0 3px rgba(44,111,176,.35);
  /* Тени */
  --shadow-sm:0 1px 2px rgba(33,28,21,.06);
  --shadow-md:0 6px 22px rgba(33,28,21,.10);
  --shadow-lg:0 18px 48px rgba(33,28,21,.18);
  /* Типографика */
  --font-serif:'Iowan Old Style','Palatino Linotype',Palatino,'Book Antiqua',Georgia,serif;
  --font-sans:-apple-system,BlinkMacSystemFont,'Segoe UI','Helvetica Neue',Arial,sans-serif;
  --font-mono:'JetBrains Mono',ui-monospace,'SF Mono',Menlo,Consolas,monospace;
  /* Геометрия */
  --radius:10px; --radius-sm:7px; --radius-lg:14px; --grid:20px; --sidebar-w:248px;
}
```

**Шрифты.** Заголовки (`h1–h4`, `.card__title`, `.modal__title`) — **serif** `var(--font-serif)`.
Тело — sans. **Все числа, коды, ID, время, цены — `var(--font-mono)`** с `font-variant-numeric: tabular-nums`;
числовые колонки таблиц выровнены вправо (`.num`). JetBrains Mono подключить (Google Fonts или self-host).

**Надписи-оверлайны** (подписи метрик, заголовки колонок) — моно, 10–11px, `letter-spacing:.08–.1em`,
`text-transform:uppercase`, цвет `--ink-500`.

## 2. Компоненты админки → `apps/admin/src/ui/*`

Поведение и состояния (полный CSS — в `tokens/admin-components.css`):

### Button (`primary` / `ghost` / `danger` + `quiet`)
- Высота 38px (sm 32 / lg 44), `--radius-sm`, `font-weight:600`.
- **primary:** фон `--coral`; hover `#cf4f39`; active `--coral-ink`; focus → `--focus-ring`.
- **ghost:** `--paper` + `1px --border-strong`; hover фон `--sand-100`.
- **danger:** текст `--coral-ink`, бордер `#E7B3A6`, hover фон `#FCEEEA`.
- `:disabled` → `opacity:.45; pointer-events:none`. Иконка слева (16px) или справа.

### DataTable
- Только чтение. `thead th`: моно-капс 10.5px, фон `--sand-100`, sticky-top.
- Строки 11px паддинг, разделитель `--sand-200`; зебра `.dt--zebra` (чётные `#FBF8F2`); hover `--sand-50`.
- Числовые ячейки `.num` — моно, вправо. Действия строки — `.iconbtn` (30×30, hover-фон `--sand-200`,
  danger-вариант hover `--coral-soft`). Низ таблицы — `.tablemeta` (счётчик строк).
- Пустое состояние — `.empty` (иконка + serif-заголовок + подпись).

### Modal (S 420 / M 560 / L 780)
- Скрим `rgba(33,28,21,.42)` + blur; **focus-trap, Esc и клик-вне закрывают** (обязательно).
- Шапка (serif-заголовок + sub + крестик), тело со скроллом, опц. футер (кнопки справа, фон `--sand-50`).
- Анимации: scrim fade .14s, тело pop .16s.

### Field (TextField / NumberField / SelectField)
- Лейбл 12.5px/600; `*` обязательности — коралл. Хинт `--ink-500` 11.5px **или** ошибка `--coral-ink`+иконка.
- Инпут 38px, бордер `--border-strong`; hover `--ink-400`; **focus** `--focus` + ring.
- `.is-error` — бордер `--coral`. NumberField — моно/вправо, опц. суффикс `RSD`. Select — кастомная стрелка.

### StatCard
- `--paper`/`--border`/`--radius`, тень sm. Подпись (моно-капс) → значение (моно, 30px) → футер с трендом.
- Тренд: `up` бирюза + ↑, `down` коралл + ↓, `flat` серый. Сетка `repeat(3,1fr)` (дашборд) / `repeat(4,1fr)` (аналитика).

### DayOfWeekPicker
- 7 toggle-кнопок Пн–Вс (40×38). Выбранная — фон `--coral`, белый текст, `aria-pressed`. `role="group"`.

### DateRangeFilter
- Два `<input type="date">` («С даты» / «По дату») в `.toolbar` (flex, `align-items:flex-end`, gap 12).

### Toast (info / success / error)
- Стек снизу-справа, тёмный фон `--ink-900`, левый бордер по тону (teal/coral/голубой).
- Вход translateX+fade .22s, уход .2s; авто-скрытие ~4.2s. `aria-live="polite"`.

### Status-теги (`tag`)
- Пилюли с точкой: `ok` (бирюза), `warn` (янтарь), `info` (песок), `coral`, `muted`. Моно 11px/600.

### API-индикатор
- `.apistat` — точка-LED (ok бирюза / down коралл) + моно-подпись. На логине и дашборде.

## 3. Каркас и экраны админки

**Каркас:** грид `248px 1fr`. Сайдбар `--sand-100` + правый бордер:
бренд-марка `BeoSand` (serif, точка-коралл) + «АДМИН-КОНСОЛЬ» (моно-капс) → меню 13 пунктов
**в 3 группах** (Расписание / Корты / Коммуникации) → подвал: переключатель языка (RU/SR/EN),
имя админа (@username), кнопка «Выйти». Активный пункт: `--paper` + коралловый левый индикатор 3px + иконка-коралл.
Бейдж счётчика на «Заявки на корт». Контент справа: serif-заголовок H1 (28px) + lead-подзаголовок (`--ink-500`).

**Логин** (`/login`): центр-карточка ≤420px на фоне с мягкими коралл/бирюза радиальными пятнами;
бренд → «Вход» → **Telegram Login Widget** (синяя `#2AABEE` кнопка) → API-индикатор + ошибка «не админ».

**Приоритетные экраны (детали смотреть в `Admin Console.html`):**
- **Обзор** — 6 StatCard (3×2) + блоки «Очередь заявок» / «Статус системы». Задаёт тон.
- **Тренировки** — тулбар (DateRange + select группы + «Сгенерировать месяц») → плотная зебра-таблица;
  в строке занятость (мини-бар `booked/cap`), статус-тег, 3 действия (добавить человека / вместимость / отменить).
  Модалка генерации: группа + месяц + **выбор корта**. Модалка вместимости с валидацией (нельзя < записанных).
- **Заявки на корт** — табы pending/confirmed/rejected/cancelled со счётчиками. Подтверждение → модалка
  выбора **свободного корта** (карточки, список от API). **Номер корта показывается только у подтверждённых.**
- **Загрузка кортов** — тепловая карта: колонки = корты, строки = 30-мин слоты 07:00–22:00.
  Ячейки: свободно (paper) / заявка (`--coral-soft`, ●) / блок (`--sand-300`, ✕). Клик по заявке → модалка.
  Легенда + выбор даты. Sticky шапка и колонка времени.
- **Рассылки** — визард 3 шага (степпер с номерами): Тип → Аудитория → Превью (число получателей + текст) → отправка.

**Остальные** (Группы, Тренеры, Уровни, Посещаемость, Клиенты, Блокировки, Аналитика, Метки) —
тот же словарь (таблицы/формы/теги/тосты). См. прототип.

## 4. Мини-апп → `apps/miniapp`

**Платформа.** Поверх `@telegram-apps/telegram-ui`: нативный вид Telegram, авто iOS/Android и светлая/тёмная темы.
**Коралл `--co:#E15B43`** кладётся поверх нативной темы: иконки плиток, активные состояния, `MainButton`, радио, бейджи.
Токены и классы — в `tokens/miniapp-theme.css` (для тёмной темы продублировать переменные под `prefers-color-scheme: dark` / `tg-theme`-классом).

**Навигация — нативная.** `BackButton` Telegram (скрыта на Home, иначе → назад). У каждого экрана-действия
ровно одна `MainButton` снизу («Записаться», «Отменить», «Отправить заявку», «Запросить тренировку»…).
На Home `MainButton` нет — это хаб. Сценарий ≤3 тапа.

**Компоненты (`apps/miniapp/src/ui/*`), привести к виду из прототипа:**
- `SlotCard` — время (700, 20px) + дата, бейдж свободных мест (бирюза / янтарь ≤2 / коралл «мест нет»),
  группа, тренер + уровень-чип, низ: цена RSD + шеврон.
- `BookingItemCard` — то же + статус-чип: записан (коралл) / был (бирюза) / не явился (янтарь) / отменён (серый).
- `Chips` + `FilterSheet` — горизонтальный скролл-ряд пилюль (Сегодня/Утро/Вечер/Есть места); активная — коралл.
- `OptionList` (`OptionRow`) — радио-строки (длительность, месяц, язык, тренер); кольцо/точка — коралл.
- `Segment` — сегмент-контрол (Предстоящие/Прошедшие).
- `StateView` — единые loading (спиннер) / empty / error / **спокойные «не-ошибки»** (тренер недоступен,
  окно листа ожидания закрылось) — отдельный спокойный тон (янтарь/нейтраль), **не красный alarm**.
- `ConfirmView` — шаг подтверждения (summary-строки + нота-хинт), переиспользуется в брони/отмене/корте.

**Экраны (см. `Mini App.html`):** Онбординг (3 шага: имя→язык→уровень), Home (6 плиток в 3 группах:
Тренировки/Корты/Аккаунт + приветственный коралл-баннер), Browse (чипы + список SlotCard → ConfirmView →
MainButton «Записаться»; занято → лист ожидания), My Bookings (сегмент + отмена), Group (список→детали→
пикер месяца→результат), Individual (список тренеров→результат), **Court** (дата 14 дней → время с бейджами
свободных кортов → длительность 1/1.5/2ч → превью цены → pending; **номер корта НЕ показывается**),
Profile (read-only + язык), Waitlist Accept (один экран-действие, исход бронь **или** спокойное «окно закрылось»).

## 5. Сквозные требования

- **Доступность:** семантика, видимые focus-кольца (`--focus-ring`), контраст, `aria-*`, `role` на toggle/tabs/radio,
  focus-trap в модалках, `aria-live` на тостах.
- **Состояния всего:** loading / empty / error у каждого списка и формы. Спокойные «не-ошибки» — отдельный тон.
- **RSD везде:** целые динары, формат `1 500`, считает сервер — UI только показывает.
- **Номер корта клиенту не виден** до подтверждения админом (мини-апп и публичные данные).
- **Три языка (RU/SR/EN)** в одной разметке: оставлять запас по длине строк, не привязывать лейаут к ширине RU.
- **Тон:** админка — десктоп, плотная, утилитарная, serif-характер; мини-апп — мобильный, нативный, лёгкий, коралл-акцент.

## 6. Порядок работ для Claude Code

1. Влить токены §1 в `apps/admin/src/ui/theme.css`; подключить JetBrains Mono. Прогнать визуальную регрессию по `Admin Console.html`.
2. Привести компоненты `apps/admin/src/ui/*` к §2 — реализовать **все** состояния (hover/focus/active/disabled/error/empty).
3. Собрать каркас (сайдбар 3-группами, lead-заголовки) и пройти по приоритетным экранам §3 (Обзор → Тренировки → Заявки → Загрузка → Рассылки), затем остальные.
4. В `apps/miniapp`: влить `tokens/miniapp-theme.css`, навесить коралл-акцент на telegram-ui, реализовать компоненты §4 и нативную навигацию (`BackButton`/`MainButton`), затем экраны.
5. Проверить сквозные требования §5: a11y, состояния, RSD-формат, скрытие номера корта, запас на i18n, тёмная тема мини-аппа.

> Источник истины по пикселям и поведению — открытые в браузере `Admin Console.html` и `Mini App.html`.
> Бизнес-логику не трогаем: значения приходят с сервера, UI их только отображает.
