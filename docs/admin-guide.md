# Памятка администратора BeoSand

Короткое руководство по текущей админ-консоли (`apps/admin`). Консоль показывает данные и отправляет
действия в API; цены, места, статусы, доступность и уведомления считает сервер.

## Вход

1. Откройте админ-консоль и страницу `/login`.
2. Войдите через Telegram Login.
3. Доступ есть у Telegram ID из `ADMIN_TELEGRAM_IDS` и у активных записей в разделе
   **Администраторы**. Если прав нет, API вернет 403.

## Меню

**Расписание:** Обзор, Группы, Тренировки, Тренеры, Администраторы, Уровни, Посещаемость, Клиенты,
Абонементы.

**Корты:** Заявки на корты, Блокировки кортов, Загрузка кортов.

**Коммуникации:** Рассылки, Аналитика, Тексты, Шаблоны уведомлений, Коннекторы.

## Базовая настройка

1. В **Уровнях** заведите уровни подготовки.
2. В **Тренерах** добавьте тренеров, их Telegram ID или username, язык и видимость для индивидуальных
   тренировок.
3. В **Администраторах** добавьте коллег. Запись можно создать по username; числовой Telegram ID
   заполняется после первого контакта, если API сможет его определить.
4. В **Группах** создайте регулярные слоты: дни недели, время, тренер, уровень, домашний корт,
   вместимость и цены в RSD.

## Расписание и клиенты

- **Тренировки** - список занятий, генерация месяца по активным группам, изменение вместимости,
  отмена, ручное добавление клиента и просмотр состава.
- **Посещаемость** - отметки `attended` и `no_show` по выбранной тренировке.
- **Клиенты** - поиск, карточки Telegram-клиентов и walk-in клиентов, контактные данные и уровень.
- **Абонементы** - месячные групповые записи; оплата меняется для всего набора неотмененных броней
  одного абонемента.

## Корты

- **Заявки на корты** - обработка клиентских запросов: подтвердить с назначением свободных кортов
  или отклонить. Клиент видит номера кортов только после подтверждения.
- **Блокировки кортов** - ручные блоки на дату и время: ремонт, турнир, служебная бронь.
- **Загрузка кортов** - дневная сетка занятости по кортам и времени.

API не даст подтвердить больше заявок, чем доступно активных кортов с учетом блокировок, групповых
занятий и уже удержанных/подтвержденных заявок.

## Коммуникации и настройки

- **Рассылки** - отправка готовых типов рассылок по выбранной аудитории.
- **Аналитика** - отчеты за период по броням, посещаемости, отменам, загрузке и рассылкам.
- **Тексты** - правки UI-надписей для RU/SR/EN поверх статического каталога.
- **Шаблоны уведомлений** - редактируемые тексты поддерживаемых уведомлений.
- **Коннекторы** - внешние интеграции, webhooks, календарные ссылки, экспорт и операционные
  настройки.

## Правила

- Деньги всегда в RSD, целыми числами.
- Не обходите серверные статусы вручную: консоль должна вызывать API-действия.
- Секреты и токены не вводятся в браузерные поля; они живут в env Railway/API.
