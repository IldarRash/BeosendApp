# Как проверить BeoSand в боте (ручное тестирование)

Пошаговая инструкция: что и в каком порядке нажимать в Telegram-боте, чтобы убедиться, что
работают оба домена — запись на **тренировки** (школа) и аренда **кортов** (Edition 2). Написано
под текущий локальный запуск (см. `local-run.md`).

> Сейчас запущено: API на `:3000`, бот (long polling), админка на `:5174` (порт `5173` был занят
> прошлым запуском, Vite сам взял следующий). Бот работает на реальном токене из `@BotFather`.

---

## 0. Что заработает сразу, а что — после подготовки

| Сценарий | Нужны данные? | Готово сейчас |
|---|---|---|
| Онбординг (`/start`) | нет | ✅ |
| Аренда корта (клиент) | нет (6 кортов в seed) | ✅ |
| Связаться с менеджером | нет | ✅ |
| Модерация заявок на корт (админ) | нет | ✅ (после шага 1) |
| Загрузка кортов (админ) | нет | ✅ (после шага 1) |
| Доступные тренировки / запись / мои записи | **да** — нужны группа и расписание | после шага 3 |
| Запись в группу на месяц | **да** | после шага 3 |
| Лист ожидания | **да** (заполненный слот) | после шага 3 |
| Менеджер `/manage`, рассылки `/broadcast`, статистика `/stats` | частично | после шага 1 (+ шаг 3 для данных) |
| Тренер `/today` | да (тренер с привязанным telegram_id) | продвинутый кейс |

В seed сейчас лежат: уровни (Beginner / Intermediate / Advanced), тренеры (Milena, Danilo) и 6 кортов.
Групп и тренировок **нет** — их создаём в шаге 3.

## 1. Стать админом (обязательно для всех админ/менеджер-сценариев)

Личность пользователя — это **числовой Telegram ID**. Админом считается тот, чей ID указан в
`ADMIN_TELEGRAM_IDS`. Сейчас там пусто ⇒ все админские действия отдают 403, а в боте даже не
показываются админские кнопки.

1. Узнайте свой ID: напишите боту **@userinfobot** (или @getmyid_bot) — он пришлёт `Id: 123456789`.
2. Впишите его в корневой `.env`:
   ```
   ADMIN_TELEGRAM_IDS=123456789
   ```
   (несколько админов — через запятую).
3. **Перезапустите API и бота** — переменные читаются один раз при старте процесса.
   (Меня можно просто попросить: «перезапусти с моим id 123456789».)

После этого в главном меню у вас появятся кнопки «🛠 Заявки на корт (админ)» и
«📊 Загрузка кортов (админ)», а команды `/manage`, `/broadcast`, `/stats` начнут пускать внутрь.

## 2. Клиентские сценарии, которые работают без подготовки

### 2.1 Онбординг
1. Откройте бота в Telegram, нажмите **Start** (или отправьте `/start`).
2. Бот спросит имя — введите любое.
3. Выберите уровень из кнопок (или «🤷 Не знаю»).
4. Должно появиться главное меню «Добро пожаловать в BeoSand 🏐».

> Нюанс: из-за повторного прогона seed уровней по 2 копии — на экране уровня может быть 6 кнопок
> (по 2 на каждый). Это косметика, на запись не влияет. См. раздел «Известные нюансы».

### 2.2 Аренда корта (полный цикл клиент → админ)
Это самый показательный сквозной сценарий — работает на текущих данных.

**Клиент:**
1. Главное меню → **🏖 Арендовать корт**.
2. Выберите дату (предлагается 7 ближайших дней).
3. Выберите время начала (показываются только часы, где есть свободные корты).
4. Выберите длительность (1 или 2 часа).
5. Бот покажет **цену в RSD** и сводку — нажмите подтвердить.
6. Ответ: «заявка отправлена». **Номер корта клиенту не показывается** — это и есть инвариант.

**Админ (тот же или другой аккаунт с админ-правами):**
1. Главное меню → **🛠 Заявки на корт (админ)**.
2. Выберите заявку → «Подтвердить» → бот покажет список **свободных** кортов на нужные часы.
3. Выберите корт — клиенту уйдёт уведомление с номером корта.
   Либо «Отклонить» — клиент получит отказ.

Что проверяем: цена считается сервером; клиент не видит и не выбирает корт; на пересекающийся час
нельзя подтвердить больше, чем есть свободных кортов.

### 2.3 Загрузка кортов (админ)
Главное меню → **📊 Загрузка кортов (админ)** → выберите дату → текстовая сетка занятости
(подтверждённые заявки + блокировки). Только чтение.

### 2.4 Связаться с менеджером
Главное меню → **ℹ️ Связаться с менеджером** → контакт менеджера из конфигурации.

## 3. Создать тестовые данные для тренировок (через API)

Чтобы заработали «Доступные тренировки», «Записаться в группу», «Мои записи» и лист ожидания,
нужны **группа** и сгенерированное на месяц **расписание**. В боте кнопок для этого нет — создаём
через API (нужны админ-права из шага 1). Команды для bash; подставьте свой admin-id.

```bash
ADMIN=123456789   # ваш Telegram ID из шага 1

# 1) узнать id уровня и тренера (берём первые из списка)
LEVEL=$(curl -s localhost:3000/levels  | python -c "import sys,json;print(json.load(sys.stdin)[0]['id'])")
TRAINER=$(curl -s localhost:3000/trainers | python -c "import sys,json;print(json.load(sys.stdin)[0]['id'])")

# 2) создать группу (вечерняя, пн/ср, 12 мест)
GROUP=$(curl -s -X POST localhost:3000/groups \
  -H "x-telegram-id: $ADMIN" -H "content-type: application/json" \
  -d "{\"name\":\"Вечерняя группа\",\"levelId\":\"$LEVEL\",\"daysOfWeek\":[1,3],
       \"startTime\":\"20:00\",\"endTime\":\"21:30\",\"trainerId\":\"$TRAINER\",
       \"capacity\":12,\"priceSingleRsd\":1500,\"priceMonthRsd\":10000}" \
  | python -c "import sys,json;print(json.load(sys.stdin)['id'])")

# 3) сгенерировать тренировки на месяц (год/месяц — например июнь 2026)
curl -s -X POST localhost:3000/trainings/generate \
  -H "x-telegram-id: $ADMIN" -H "content-type: application/json" \
  -d "{\"groupId\":\"$GROUP\",\"year\":2026,\"month\":6}"

# проверка: список тренировок месяца
curl -s "localhost:3000/trainings?from=2026-06-01&to=2026-06-30" -H "x-telegram-id: $ADMIN"
```

> `daysOfWeek` — числа 1–7 (1 = понедельник … 7 = воскресенье). Цены — целые RSD.
> Если `python` нет — подставьте id уровня/тренера руками из `curl localhost:3000/levels` и `/trainers`.

После генерации появятся конкретные тренировки на оставшиеся пн/ср июня — их и будем бронировать.

## 4. Сценарии записи на тренировки (после шага 3)

### 4.1 Доступные тренировки и разовая запись
1. Главное меню → **🏐 Доступные тренировки**.
2. Видны только **бронируемые** слоты (статус `open`, есть свободные места), с ценой за разовое.
3. (Опционально) поиграйте с фильтрами-чипами: день недели / время суток / тренер / уровень.
4. Выберите слот → **Записаться** → карточка подтверждения → **Подтвердить запись**.
5. Откройте **📋 Мои записи** — запись должна быть там.

### 4.2 Свободные места сегодня
Главное меню → **📅 Свободные места сегодня** — слоты только на сегодняшнюю дату (если есть).

### 4.3 Отмена записи
**📋 Мои записи** → у записи кнопка отмены → «вы уверены?» → подтвердить. Запись уходит в `cancelled`,
место освобождается (статус тренировки может вернуться `full → open`).

### 4.4 Запись в группу на месяц
1. Главное меню → **👥 Записаться в группу**.
2. Выберите группу → выберите месяц → подтвердите.
3. Создаётся **по одной записи на каждую тренировку месяца** (одна пачка, связана общим
   `group_subscription_id`).
4. Проверка инварианта: отмените **одну** дату в «Мои записи» — остальные даты месяца должны
   остаться. Отмена одной даты не должна снести всю подписку.

### 4.5 Лист ожидания
1. Заполните слот (запишите столько клиентов, сколько вместимость — или временно уменьшите
   вместимость через `/manage`, см. ниже).
2. На заполненном слоте клиенту предлагается **встать в лист ожидания**, а не обычная запись.
3. Когда кто-то отменяет запись — первому в листе ожидания приходит push с предложением занять место.

## 5. Менеджерские сценарии

### 5.1 Консоль менеджера — `/manage`
Отправьте `/manage` (нужны админ-права). Кнопки:
- **📊 Обзор заполненности** — список тренировок на 30 дней с `записано/вместимость` и статусом.
- **🔢 Изменить вместимость** — выбрать тренировку → новое число мест. Нельзя задать меньше уже
  записанных (бот покажет соответствующее сообщение); сервер пересчитает `open/full`.
- **🚫 Отменить тренировку** — выбрать → подтвердить. Записанные клиенты уведомляются, места
  освобождаются, статус → `cancelled`.
- **📨 Рассылки** и **📈 Сводка по школе** — см. ниже.

### 5.2 Рассылки свободных слотов — `/broadcast`
`/broadcast` → выбрать тип → выбрать аудиторию (все / по уровню / активные / «уснувшие») →
предпросмотр (с количеством получателей и кнопками-диплинками на запись) → отправить.

### 5.3 Аналитика — `/stats`
`/stats` → сводка по школе (популярные слоты, заполняемость, отмены и т.п.), только чтение.

## 6. Сценарий тренера — `/today` (продвинутый)

`/today` доступен пользователю, чей `telegram_id` привязан к тренеру (`trainers.telegram_id`). У
seed-тренеров привязки нет, поэтому для проверки нужно проставить ваш ID тренеру в БД, например:

```bash
# поставить ваш telegram_id первому тренеру (через psql в контейнере)
docker exec -it beosand-postgres psql -U beosand -d beosand \
  -c "update trainers set telegram_id=123456789 where name='Milena' limit 1;"
```

Затем `/today` → список сегодняшних тренировок → состав группы (roster) → отметка посещаемости
(пришёл / не пришёл).

## 7. Быстрая проверка «всё живо» (curl)

```bash
curl -s localhost:3000/health                                   # {"status":"ok",...}
curl -s localhost:3000/levels                                   # уровни из БД
curl -s "localhost:3000/court-requests/availability?date=2026-06-10"  # 13 часов × 6 кортов
curl -s -o /dev/null -w "%{http_code}\n" localhost:3000/courts                       # 400 (без заголовка)
curl -s -o /dev/null -w "%{http_code}\n" -H "x-telegram-id: 123" localhost:3000/courts  # 403 (не админ)
curl -s -o /dev/null -w "%{http_code}\n" localhost:5174         # 200 (админка)
```

## 8. Известные нюансы (не баги вашей проверки)

- **Дубликаты seed.** Уровней и тренеров по 2 копии — seed прогнался дважды (на прошлом запуске и
  сейчас), а у `levels`/`trainers` нет уникального ключа по имени (у `courts` есть, их ровно 6). На
  экранах будут лишние кнопки уровней/тренеров. Это не влияет на запись; лечится либо чисткой БД
  (`pnpm db:down -v` → `up` → `migrate` → `seed`), либо добавлением уникального ограничения
  (отдельная задача со схемой+миграцией).
- **Админка на `:5174`, а не `:5173`.** Порт 5173 занят прошлым процессом Vite. Можно остановить тот
  процесс и перезапустить админку, либо просто открывать `:5174`.
- **Переменные окружения читаются при старте.** Любая правка `.env` (в т.ч. `ADMIN_TELEGRAM_IDS`)
  требует перезапуска API и бота.

## 9. Логи и перезапуск

- Логи запущенных процессов: `/tmp/beosand-api.log`, `/tmp/beosand-bot.log`, `/tmp/beosand-admin.log`.
- Полная остановка: `docker compose down` (данные сохраняются) или `docker compose down -v` (с
  очисткой), плюс остановить процессы API/бот/админка.
- Чистый перезапуск стека описан в `docs/product/features/local-run.md`.
