# BeoSand — план дизайна: Админ-консоль и Мини-апп

Документ для дизайнера. Описывает **что есть**, **что нужно нарисовать** и **в каких рамках**.
Две поверхности продукта:

- **Админ-консоль** (`apps/admin`) — веб, React + Vite. Рабочий инструмент менеджера/администратора школы.
- **Мини-апп** (`apps/miniapp`) — Telegram Mini App, React. Клиентский интерфейс внутри Telegram.

Обе — **только слой взаимодействия**: вся бизнес-логика (цены, доступность, вместимость, статусы)
считается на сервере. Дизайн ничего не «вычисляет», он показывает готовые значения и собирает действия.

Язык интерфейса: **русский** (сербский и английский — зеркальные переводы, та же структура экранов).
Деньги — всегда **RSD** (целые динары), формат с разделением разрядов: `1 500`.

## Часть 1. Админ-консоль (`apps/admin`)

### 1.1. Общий каркас

- **Левый сайдбар** (≈248px): бренд-марка «BeoSand» + подпись «АДМИН-КОНСОЛЬ», вертикальное меню (13 пунктов),
  внизу — переключатель языка, имя залогиненного админа (@username), кнопка «Выйти».
- **Основная область** справа: заголовок страницы + текст-подзаголовок (lead), затем контент.
- Десктоп-first, плотная информационная сетка. Это инструмент для работы, не лендинг.

### 1.2. Текущий визуальный язык (что уже заложено в `src/ui/theme.css`)

Не «генерик-AI-эстетика». Утилитарно, чисто, с характером:

- **Палитра:** песочные фоны (`sand-50/100/200`), чернильный текст (`ink-900/700/500`),
  акцент — **коралловый** (`coral` + тёмный `coral-ink`), успех — **бирюзовый** (`teal`),
  бумага (белый), тонкие линии-границы, мягкая тень.
- **Типографика:** заголовки — **serif** (Iowan Old Style / Palatino), тело — sans-serif,
  цифры/коды — **моноширинный** (JetBrains Mono). Числовые колонки в таблицах — моно и выравнивание вправо.
- **Геометрия:** радиус `10px`, базовый отступ-сетки `20px`.

**Задача дизайнера:** довести этот язык до целостной системы — состояния компонентов (hover/focus/disabled/active),
плотность таблиц, иерархия заголовков, отступы, фокус-кольца (доступность обязательна: семантика, контраст, `aria`).

### 1.3. Дизайн-система — компоненты (существуют, нужно отрисовать состояния)

| Компонент | Что это | Нужно от дизайнера |
|---|---|---|
| **Button** | варианты `primary` (коралл), `ghost` (контур), `danger` | все состояния, размеры, иконка+текст |
| **DataTable** | таблица только для чтения, числовые колонки моно/вправо | плотность строк, зебра/линии, пустое состояние, сортировка-вид |
| **Modal** | диалог (focus-trap, Esc/клик вне), опциональный футер с кнопками | размеры (S/M/L), шапка, скролл тела, футер действий |
| **Field** | `TextField` / `NumberField` / `SelectField` + подпись, хинт, ошибка | состояния валидации, hint vs error, disabled |
| **StatCard** | метрика: подпись (моно-капс), крупное значение (моно), хинт | сетка карточек, акцент на тренде |
| **DayOfWeekPicker** | мультивыбор дней недели (Пн–Вс), toggle-кнопки | выбранное/невыбранное, группа |
| **DateRangeFilter** | два поля «с / по» (ISO-даты) | компоновка в тулбаре |
| **Toast** | стек уведомлений снизу-справа, тона `info/success/error` | вид по тонам, анимация появления/ухода |
| **Status-теги (`tag`)** | пилюли статусов `ok/warn/info` | цветовая семантика статусов |

### 1.4. Экраны (13 разделов меню + логин)

**Логин** (`/login`) — публичный. Центрированная карточка (≤420px): бренд, заголовок «Вход»,
официальный **Telegram Login Widget**, индикатор состояния API (зелёная/красная точка), ошибка «не админ».

|---|---|---|---|---|
| 1 | **Обзор (Dashboard)** | `/` | 6 карточек метрик (брони, заполняемость, активные клиенты, % отмен, % неявок, топ-слот) + статус API | Главная — задаёт тон. Иерархия карточек |

| 7 | **Клиенты** | `/clients` | Поиск (имя/@username) + фильтр статуса, таблица. Отдельная форма онбординга (Telegram ID, имя, username, уровень) | Поиск + список |

**Приоритет для дизайна:** `1 Обзор` (тон), `3 Тренировки` (плотность + действия),

## Часть 2. Мини-апп (`apps/miniapp`)

### 2.1. Платформа и правила

- Это **Telegram Mini App** — живёт внутри Telegram, на мобильном. Используется библиотека
  `@telegram-apps/telegram-ui`: нативный вид Telegram, авто-адаптация **iOS (HIG) / Android (Material)**
  и **светлая/тёмная** темы от системы.
- **Акцент BeoSand — коралловый** поверх нативной темы (иконки меню, активные состояния, главные кнопки).
- **Навигация — нативная:** `BackButton` Telegram (на Home скрыта, на остальных — назад к Home).
  Браузерной кнопки «назад» нет. Каждый экран действия владеет одной нативной `MainButton` внизу
  (например «Записаться», «Отменить», «Отправить заявку»). На Home `MainButton` нет — это хаб.
- Журналы коротки: **≤3 тапа** на сценарий (открыл → выбрал → подтвердил).
- **Клиент никогда не видит номер корта** до подтверждения админом. Цены — RSD, считает сервер, показ только.

### 2.2. Дизайн-система мини-аппа (существует)

Свои переиспользуемые компоненты в `apps/miniapp/src/ui/`:

- `StateView` — единые состояния загрузки / ошибки / пустоты.
- `ConfirmView` — шаг подтверждения (bottom-sheet / детализация). Переиспользуется в брони, отмене и т.д.
- `SlotCard` — карточка слота тренировки (дата, время, тренер, уровень, свободные места, цена).
- `BookingItemCard` — строка брони в «Моих записях».
- `OptionList` — одиночный выбор (длительность, месяц, фильтры).
- `FilterSheet` + `Chips` — фильтры просмотра (день недели, время суток, тренер, уровень, «Сегодня»).
- `FallbackButton` / `SecondaryButton` — вторичные кнопки и фолбэк вне Telegram.
- `icons.tsx` — коралловые иконки меню (browse, myBookings, group, individual, court, profile).

**Задача дизайнера:** привести их к единому виду в рамках Telegram-UI, не ломая нативность;
проработать карточки (`SlotCard`, `BookingItemCard`), фильтры (`Chips`/`FilterSheet`), пустые/ошибочные
состояния (`StateView`), и коралловый акцент так, чтобы он читался и в светлой, и в тёмной теме.

### 2.3. Экраны и сценарии (8 экранов, хаб-и-спицы)

**Онбординг** (только для нового пользователя, до Home): мастер из 3 шагов —
имя (предзаполнено из Telegram) → язык (RU/SR/EN, переключает UI вживую) → уровень (необязательно, «решу позже»).

| Экран | Что видит пользователь | Сценарий | Внимание дизайнера |
|---|---|---|---|
| **Home** | Хаб: 6 плиток-журналов в 3 группах — **Тренировки / Корты / Аккаунт** | Тап по плитке → переход в журнал | Главный экран. Группировка, иконки, иерархия плиток |
| **Browse (просмотр и запись)** | Список доступных слотов (карточки) + фильтры (день, время суток, тренер, уровень, «Сегодня») | Тап по слоту → `ConfirmView` → MainButton «Записаться». Если занято (409) → предложить лист ожидания | Карточка слота + панель фильтров. Ключевой экран |
| **My Bookings (мои записи)** | Сегмент «Предстоящие / Прошедшие». Тренер, уровень, дата, время, статус/исход | Переключение вкладок; «Отменить» на предстоящей → подтверждение → обновление | Сегмент-контрол, статус-чипы, исходы (attended/no_show/cancelled) |
| **Group (абонемент на месяц)** | Список групп → детали (дни, время, тренер, цена/мес RSD) → выбор месяца (текущий/следующий) → результат | Группа → месяц → MainButton → результат (создано N, пропущено дат) | Список → пикер месяца → экран результата |
| **Individual (запрос тренера)** | Список тренеров (имя + тип main/guest) → подтверждение | Тренер → MainButton «Запросить тренировку» → результат (доставлено / «тренер недоступен» — спокойное состояние, не ошибка) | Список + спокойные исходы |
| **Court (заявка на корт)** | Поток вперёд: дата (14 дней) → время (предложенные старты + счётчик свободных кортов) → длительность (1 / 1.5 / 2 ч) → превью цены RSD → pending | Последовательные тапы → MainButton «Отправить заявку» → «в обработке» | Многошаговый пикер. **Без номера корта.** Бейджи свободных кортов |
| **Profile (профиль)** | Только чтение: имя, уровень (или плейсхолдер), @username. Редактируемо: язык | Смена языка → оптимистично + сохранение | Простой read-only + один контрол |
| **Waitlist Accept** | «Освободилось место» + контекст | MainButton «Подтвердить» → бронь или спокойное «окно закрылось» (если 409) | Входит по deep-link. Один экран-действие |

**Приоритет для дизайна:** `Home` (хаб, плитки, иконки), `Browse` (карточка слота + фильтры),
`My Bookings` (сегмент + чипы), `Court` (многошаговый пикер). Онбординг — первое впечатление.

## Часть 3. Сквозные требования для обеих поверхностей

- **Доступность не опциональна:** семантика, фокус, контраст, `aria-*`. Фокус-кольца видимы.
- **Состояния всего:** loading / empty / error для каждого списка и формы. Спокойные «не-ошибки»
  (тренер недоступен, окно листа ожидания закрылось) — отдельный тон, не красный alarm.
- **RSD везде:** целые динары, формат `1 500`, считается сервером — дизайн только показывает.
- **Номер корта клиенту не показывается** до подтверждения админом (мини-апп и публичные данные).
- **Три языка** в одной разметке: оставляйте запас на длину строк (RU/SR/EN), не привязывайте лейаут к ширине русского текста.
- **Разделение тона:** админка — десктоп, плотная, утилитарная, с serif-характером; мини-апп — мобильный,
  нативный Telegram, лёгкий, коралловый акцент.
